하루 작업 루틴

시점명령어설명목적
아침 시작git pull
gitx status		원격 최신 코드/상태 점검시작점 세팅
작업 중gitx q빠른 백업(자동 메시지)안전 스냅샷
작업 중gitx save "메시지"정상 백업(메시지 기록)의미 기록
중간 점검gitx log 10최근 커밋 확인흐름 파악
기능 완성gitx series-bump버전 증가(patch/minor/major)이정표
마감 전gitx status잔여 변경/상태 점검정리
퇴근git push원격 동기화안전 보관

버전 관리 전략

라인
  • main: 항상 배포 가능한 안정 라인
  • series/N.x: 메이저 N 개발 줄기
  • feat/*: 기능 단위 실험/작업
키 포인트
  • VERSION 파일로 현재 버전 관리
  • git tag vX.Y.Z로 배포/스냅샷
  • 안정화되면 main에 승격

예제 시나리오

# 빠른 저장
gitx q
# 의미 기록
gitx save "feat: 로또 추천 개선"
# 서브 릴리스
gitx series-tag
# 메인 승격
gitx series-promote
# 다음 메이저 시작
gitx series-next
